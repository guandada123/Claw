# Prompt 缓存优化 Skill

> 用于Claw项目WorkBuddy自动化的DeepSeek缓存命中率优化
> 目标：95% → 98%+，预计每日节省¥5-8

## 触发场景
- 用户要求优化API成本、缓存命中率、提示词
- 新创建的自动化需要prompt模板
- 发现输入缓存未命中比例偏高

## 步骤

### 1. 识别角色
Claw项目有3个独立角色，各自有固定base prompt：
- **模拟炒股** — AI全权操盘手，¥30,000，禁创科北ST
- **投资助理** — 用户顾问，仅建议权，不碰simulation/
- **美股监控** — 美股分析师，仅监控不交易

### 2. 按模型类型选择prompt风格
| 模型 | 风格 | 固定前缀长度 | 适用 |
|------|------|------------|------|
| Flash | 指令式 | ≤100字 | P0高频任务 |
| V4-Pro | 结构化框架 | ≤300字 | P2中频深度分析 |
| Reasoner | 引导式推理 | ≤200字 | P1/P3判断型 |

### 3. 结构模板
```
[角色] {角色定义} | {关键约束}
[权限] {权限范围}
[输出] {输出格式}
━━━━━━━━━━━ 固定指令 ▸ 缓存命中区 ━━━━━━━━━━━
{执行步骤（纯静态指令）}
{输出模板（固定格式）}
━━━━━━━━━━━ 动态数据 ▸ 不命中区 ━━━━━━━━━━━
{本次运行的变量}
```

### 4. 验证
- 检查固定前缀中**没有**时间、价格等动态数据
- 检查角色base与自动化功能匹配
- 检查输出格式模板完整
