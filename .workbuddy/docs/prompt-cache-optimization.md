# Prompt 缓存优化手册 v1.0

> 目标：将 DeepSeek 缓存命中率从 ~95% 提升至 98%+
> 原理：固定前缀 ≈ 命中缓存（¥0.025/M），动态后缀走未命中（¥3/M）
> 价差：120倍

## 二、按模型类型的 Prompt 策略

### DeepSeek V4 Flash — 轻量快

```
[策略核心] 指令式 | ≤100字固定前缀 | 少而精
[适用] P0高频自动化（盘中监控/问答/综合监控）

=== 固定前缀（缓存命中）===
{角色Base} | 模型:Flash
指令：
1. {操作A}
2. {操作B}
3. {操作C}

=== 动态数据（不命中）===
[time] {当前时间}
[prices] {实时价格}
```

### DeepSeek V4 Pro — 强指令 + 结构化

```
[策略核心] 结构化框架 | ≤300字固定前缀 | 完整的指令定义
[适用] P2中频且需要深度分析的自动化（复盘/总结/选股）

=== 固定前缀（缓存命中）===
{角色Base} | 模型:V4-Pro

## 分析要求
1. 牛熊辩论框架（看多/看空各至少3条）
2. 四维评分（消息/技术/基本/资金各0-10）
3. 风控检查（仓位/行业/大盘/止损）
4. 结论先行（代码+价位+操作）

## 输出格式
{固定模板}

=== 动态数据（不命中）===
[time] {当前时间}
[portfolio] {持仓数据}
[market] {行情数据}
```

### DeepSeek Reasoner — 思考链

```
[策略核心] 引导式推理 | ≤200字固定前缀 | 强调推理过程
[适用] P1/P3 需要市场判断的分析任务

=== 固定前缀（缓存命中）===
{角色Base} | 模型:Reasoner

## 推理要求
1. 先给出整体判断（一句话结论）
2. 列出推理链条（逐步推导）
3. 给出具体建议和价位

=== 动态数据（不命中）===
[time] {当前时间}
[data] {分析数据}
```

## 四、注意事项

1. **固定前缀中绝对不能出现动态数据**（时间、价格、持仓等）
2. **不同角色的前缀不能混用**（模拟炒股 ≠ 投资助理）
3. **切换模型类型时需重写前缀风格**（Flash式 ≠ Pro式 ≠ Reasoner式）
4. **Reasoner 的 thinking 会让 system prompt 的缓存收益打折扣**（think token 也在消耗），所以高频任务应当用 Flash
5. 优化后请观察 2-3 天的 `prompt_cache_hit_tokens` 和 `prompt_cache_miss_tokens` 指标
