## 1. 四层模型分级

| 层级 | 代表模型 | 输入成本 | 输出成本 | 适用场景 |
|------|---------|---------|---------|---------|
| 🟢 Flash | deepseek-v4-flash | ¥0.5/万Token | ¥1.5/万Token | 高频简单任务、模板化输出、数据聚合 |
| 🟡 Pro | deepseek-v4-pro | ¥4.0/万Token | ¥12.0/万Token | 中等复杂度分析、策略开发、个股深度 |
| 🔴 旗舰 | GPT-5 / Claude | ¥18-36/万Token | ¥72-180/万Token | 架构审查、核心策略、复杂调试（极少用） |
| ⚪ 本地 | Qwen2.5-7B/14B (Ollama) | ¥0 | ¥0 | 离线推理、隐私数据、连续对话（Mac Mini M4） |

### 选择原则（含 LOCAL 层）

```
极简任务 → LOCAL（心跳/巡检/简单检查，零成本）
分析任务 → Pro（保留质量但控制成本）
高价值/低频 → 旗舰（每月 1-2 次足矣）
```

### 定价对比

```
Flash  ¥0.5/万  ──────────────────── █
Pro    ¥4.0/万  ──────── ████████   (8x Flash)
GPT-5  ¥18/万   ── ████████████████ ████████████████████████████  (36x Flash)
Claude ¥21.6/万 ── ████████████████ ████████████████████████████████  (43x Flash)
```

**核心原则**: 能走本地走本地，能走 Flash 绝不用 Pro，能用 Pro 绝不用旗舰。

## 2. 路由决策流程

```
用户输入 / 任务描述
        │
        ▼
  ┌─────────────────┐
  │  1. 强制层级？   │ ← force_tier 参数可绕过
  └────────┬────────┘
           │ 否
           ▼
  ┌─────────────────┐
  │  2. 旗舰信号？   │ ← 匹配 PREMIUM_SIGNALS
  └────────┬────────┘
           │ 否
           ▼
  ┌─────────────────┐
  │  3. LOCAL 可用？  │ ← Ollama 运行中且匹配 LOCAL 规则
  └────────┬────────┘
           │ 否
           ▼
  ┌─────────────────┐
  │  4. Flash 匹配   │ ← 匹配 ROUTING_RULES[FLASH]
  └────────┬────────┘
           │ 否
           ▼
  ┌─────────────────┐
  │  5. Pro 匹配     │ ← 匹配 ROUTING_RULES[PRO]
  └────────┬────────┘
           │ 否
           ▼
  ┌─────────────────┐
  │  6. 默认 Pro     │ ← 默认保守
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │  7. 预算降级     │ ← budget_guard 覆盖（LOCAL 不受影响）
  └────────┬────────┘
           │
           ▼
     返回模型决策
```

## 4. 预算降级策略

### 预算层级

```
已用 ¥0 ─── ¥200 ─── ¥280 ─── ¥350 ─── ¥400
  │         │         │         │         │
  │ FULL    │ NORMAL  │ FLASH_  │ FLASH_  │ HARD
  │         │         │ PREFER  │ ONLY    │ LIMIT
  │         │         │         │         │
  │ 任意    │ 正常    │ Pro→    │ 全部→   │ 停止
  │ 模型    │ 调度    │ Flash   │ Flash   │ 调用
```

### 降级规则

| 预算层级 | 触发条件 | 行为 | 安全措施 |
|---------|---------|------|---------|
| FULL | 已用 < ¥200 | 正常使用任何模型 | 无 |
| NORMAL | ¥200 ≤ 已用 < ¥280 | 正常调度，标记谨慎 | 推送提醒 |
| FLASH_PREFERRED | ¥280 ≤ 已用 < ¥350 | Pro 任务降级为 Flash | 旗舰需 critical 优先级 |
| FLASH_ONLY | ¥350 ≤ 已用 ≤ ¥400 | 全部降级为 Flash | 旗舰需人工确认 |
| HARD_LIMIT | 已用 > ¥400 | **停止所有非关键调用** | 仅允许 critical |

### 单次调用拦截

```
单次预估成本 > ¥5 → 需要确认
```

## 5. Fallback 链

### 优先级

```
1. Claude Sonnet 4.5 (旗舰首选)  → timeout 30s
2. GPT-5 (旗舰后备)              → timeout 30s
3. DeepSeek V4 Pro (降级)        → timeout 20s
4. DeepSeek V4 Flash (兜底)      → timeout 15s
```

### 跨供应商策略

- DeepSeek 不可用 → 自动尝试 CatRouter 中转的旗舰模型
- CatRouter 不可用 → 降级到 Flash 直连
- 全部不可用 → 返回错误并告警

### 代码集成

```python
from router import call_with_fallback, get_model
from budget_guard import check_budget_status

budget = check_budget_status()
result = call_with_fallback(
    prompt="分析000001的技术形态",
    task_type="single_analysis",
    budget_status=budget
)
```

## 7. 成本估算方法

### 单次调用成本

```
成本 = (输入Token × 输入单价 + 输出Token × 输出单价) / 10000
```

### 日均成本

```
日均成本 = Σ(每次调用成本) + 缓存节省(-30~60%)
```

### 月度预算

```
预算 = ¥400/月
锁定线 = ¥350 (87.5%)
日警告 = ¥25/天 (6.25%)
单次上限 = ¥5
```

### 优化效果

```
优化前预估: ¥265-405/月  (多模型混合)
优化后预估: ¥100-180/月  (90% Flash + 关键任务保留质量)
节省幅度:   50-55%
```

## 8. QA 与常见问题

### Q: 降级后质量会下降吗？

**会，但可接受。** 从 GLM/Hy3/Reasoner 降级到 Flash 后：
- 模板输出、数据聚合、格式化内容 — **无明显差异**
- 推理深度、分析连贯性 — **略有下降**（但非关键任务可接受）
- 智能选股、下周前瞻等深度任务 — **保持 Kimi，未降级**

### Q: 遇到质量问题怎么办？

1. 确认是模型质量问题（非数据源问题）
2. 在 `routing-rules.md` 中查到该自动化的推荐模型
3. 暂时恢复原模型，标记为"需保留原模型"
4. 联系 Senior Developer 调整路由规则

### Q: 新创建的自动化用什么模型？

**优先使用路由判断**: 将任务描述输入 `router.get_model()` 自动选择。

如果路由判断不符合预期（如希望用Pro但返回了Flash）:
1. 检查信号词是否准确描述任务
2. 在 Prompt 中添加 `【需要深度分析】` 等关键词
3. 如果仍不合适，手动指定 `force_tier=ModelTier.PRO`

### Q: Flash 成本到底有多低？

以盘中监控为例（每小时1次，每次~2000 Token）：
- 用 Flash: **¥0.5 × 2000/10000 = ¥0.1/次** → 每天8次 = ¥0.8/天 → **¥24/月**
- 用 Pro: **¥4 × 2000/10000 = ¥0.8/次** → 每天8次 = ¥6.4/天 → **¥192/月**

**每月节省: ¥168/月**，而且输出质量对模板化任务几乎无差异。

### Q: 什么时候应该降低模型层级？

| 场景 | 做法 | 节省 |
|------|------|------|
| 任务是"格式化输出" | Flash | 8x vs Pro |
| 任务需要"深度分析" | Pro | 保留质量 |
| 任务有"架构"关键词 | 旗舰 | 仅高价值时 |
| 预算剩余<¥50 | 全部 Flash | 确保不超预算 |
