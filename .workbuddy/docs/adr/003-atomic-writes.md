# ADR-003: 原子 JSON 写入模式

## 状态
已采纳（2026-06-12）

## 背景
Claw 项目中持仓、选股、交易记录等关键数据通过 `json.dump()` 直接写入文件。在以下场景存在数据损坏风险：
- 进程崩溃（写入到一半）
- 磁盘满（文件截断）
- 无写入前校验

## 决策
使用 `AtomicJSONWriter` 原子写入模式：
1. 写入临时文件（`.json.tmp`）
2. 写入后校验可读性（`json.load` 验证）
3. `os.replace()` 原子 rename 替换原文件
4. 写入前自动备份（保留最近 N 份）

## 后果
- 正向：100% 杜绝写入中断导致的数据损坏
- 正向：自动备份机制（最多 10 份历史）支持数据恢复
- 负向：每次写入多一次 `rename` 系统调用 + 磁盘写入（可忽略的性能代价）
- 注：`max_backups=0` 可关闭备份，适用于临时/非关键文件
