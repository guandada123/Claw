## 优化原则

1. **不降级出质量问题的任务** — Kimi/GLM保留深度分析和报告任务
2. **高频任务优先切Flash** — 盘中监控类已经是Flash，保持
3. **Pro → Flash** — 消息汇总类、模板化报告，无深度推理需求
4. **旗舰 → Pro/本地** — 架构审查类改为人工触发的本地Qwen2.5
5. **新任务走路由** — 所有新建自动化必须经router.py判断模型层级
