# 研究严谨性前缀（Research Rigor Prefix）

> 本前缀源自 AI Berkshire 中文学习资料包（xbtlin/ai-berkshire, MIT License）的
> "反面证据 + 数据溯源"约束句，经 Claw 适配后作为所有投资研究的统一前置护栏。
> 任何公司研究 / 财报分析 / 早报信号 / 选股建议，**生成前先把这段要求附加到任务末尾**。

---

把以下要求附在任务末尾，可明显降低"看起来很专业但不可核验"的风险：

```
研究截止时间为今天。关键数字必须注明报告期、币种、单位和来源链接；
优先使用公司公告、交易所文件、监管披露和财报原文；
至少列出三条反面证据和可能证伪结论的条件；
不确定的数据明确写"未知"，不得估算后伪装成事实；
区分事实、推断和观点；不提供个性化买卖建议。
```

## 强制清单（逐条自检，缺一项则结论不可信）

- [ ] **溯源**：每个关键数字都标了报告期 + 币种 + 单位 + 来源链接
- [ ] **一手优先**：公司公告 / 交易所 / 监管披露 / 财报原文 > 二手转述
- [ ] **反面证据 ≥ 3 条**：明确列出可能证伪结论的条件（不是只讲利好）
- [ ] **未知不猜**：数据不足处写"未知"，不估算伪装成事实
- [ ] **三类分离**：事实 / 推断 / 观点 明确标注，不混为一谈
- [ ] **不荐个股**：不给个性化买卖建议（买卖决策过 `validate_constraints.py` 硬约束）

## 与 Claw 现有护栏的衔接

| 本前缀要求 | Claw 对应机制 |
|---|---|
| 数据源可追溯 | `source-resolution.md` 规则 + `westockdata` / `a-stock-data` 多源校验 |
| 反面证据 ≥ 3 | `thesis-tracker` 红线清单 + `memo-builder` 证伪条件 |
| 不荐个股 / 整手 | `validate_constraints.py`（整手 / 板块 / 止损铁律） |
| AI 偏见自觉 | 研究报告首段标注"事实 / 推断 / 观点"与信心水平 |

> 用法：在调用 `investment-research` / `earnings-analysis` / `memo-builder` /
> `thesis-tracker` / 早报生成 时，将上方代码块内容粘贴到用户 prompt 末尾即可。
